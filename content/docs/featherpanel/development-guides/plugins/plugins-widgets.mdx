# Plugin Widgets

Plugin widgets allow you to inject custom HTML content into existing FeatherPanel pages at specific locations. Unlike plugin pages (which create entirely new routes), widgets integrate seamlessly into the panel's existing interface, enabling you to enhance functionality without creating separate pages.

## What Are Widgets?

Widgets are HTML components that can be embedded into FeatherPanel's native pages at predefined locations. They're perfect for:

-   **Information displays** - Show additional server metrics, user statistics, or custom data
-   **Quick actions** - Add buttons or shortcuts for common operations
-   **Notifications** - Display important alerts or messages
-   **Charts and graphs** - Visualize data on dashboard or server pages
-   **Integration panels** - Display content from external services

Widgets automatically receive context information (like server UUID or user UUID) and can make API calls to your plugin's backend endpoints.

## Widget vs. Plugin Pages

| Feature         | Widgets                              | Plugin Pages                 |
| --------------- | ------------------------------------ | ---------------------------- |
| **Integration** | Embedded in existing pages           | Standalone pages with routes |
| **Location**    | Predefined locations on native pages | Custom routes via sidebar    |
| **Use Case**    | Enhance existing functionality       | Create new interfaces        |
| **Navigation**  | No navigation needed                 | Requires sidebar entry       |
| **Context**     | Automatic (server/user UUID)         | URL query parameters         |

Use **widgets** when you want to add content to existing pages. Use **plugin pages** when you need a dedicated interface for your plugin.

## Creating Widgets

### File Structure

Widgets are HTML files stored in the `Frontend/Components/` directory, alongside plugin pages:

```
storage/addons/my_plugin/
├── Frontend/
│   ├── index.js              # Plugin JavaScript initialization
│   ├── index.css             # Plugin styles
│   ├── sidebar.json          # Sidebar configuration (for pages)
│   └── Components/           # Components directory
│       ├── main-page.html    # Plugin page (if using pages)
│       └── server-console-widget.html  # Widget component
```

### Widget Configuration

Widgets are configured using JSON files. Create a `widgets.json` file in your plugin's `Frontend/` directory:

```json
{
    "id": "my-plugin-server-info",
    "component": "server-console-widget.html",
    "enabled": true,
    "priority": 100,
    "page": "server-console",
    "location": "under-server-info-cards",
    "size": "full"
}
```

#### Configuration Properties

-   **`id`** (required): Unique identifier for the widget within your plugin
-   **`component`** (required): Path to HTML file in `Components/` directory (e.g., `"widget.html"`)
-   **`page`** (required): Target page identifier (see [Supported Pages](#supported-pages))
-   **`location`** (required): Widget placement location (see location sections below)
-   **`enabled`** (optional): Enable or disable the widget (default: `true`)
-   **`priority`** (optional): Rendering order - higher values render first (default: `100`)
-   **`size`** (optional): Widget width - `"full"`, `"half"`, `"third"`, or `"quarter"` (default: `"full"`)

#### Multiple Widgets

You can define multiple widgets in a single array:

```json
[
    {
        "id": "widget-1",
        "component": "widget1.html",
        "page": "server-console",
        "location": "under-server-info-cards",
        "size": "half",
        "priority": 100
    },
    {
        "id": "widget-2",
        "component": "widget2.html",
        "page": "server-console",
        "location": "under-server-info-cards",
        "size": "half",
        "priority": 90
    }
]
```

### Widget Sizing

Widgets support flexible sizing to create responsive layouts:

-   **`"full"`** - Full width widget (default)

    -   Best for: Info cards, charts, tables, forms
    -   Renders: 1 widget per row

-   **`"half"`** - Half width widget

    -   Best for: Side-by-side information displays
    -   Renders: 2 widgets per row on desktop

-   **`"third"`** - One-third width widget

    -   Best for: Statistics cards, small charts
    -   Renders: 3 widgets per row on desktop

-   **`"quarter"`** - One-quarter width widget
    -   Best for: Quick stats, icons with labels
    -   Renders: 4 widgets per row on desktop

**Note:** All widgets automatically stack vertically on mobile screens for optimal mobile experience.

### Creating Widget HTML

Widget HTML files follow the same structure as plugin pages, but are typically more focused since they're embedded in existing pages:

```html
<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8" />
        <meta name="viewport" content="width=device-width, initial-scale=1.0" />
        <title>Server Info Widget</title>
        <style>
            .widget-container {
                background: rgba(18, 19, 21, 0.8);
                border-radius: 0.75rem;
                padding: 1.5rem;
                border: 1px solid rgba(255, 255, 255, 0.07);
                margin-bottom: 1rem;
            }

            .widget-title {
                font-size: 1.125rem;
                font-weight: 600;
                margin-bottom: 0.75rem;
                color: #fafafa;
            }

            .widget-content {
                color: #e5e5e5;
            }
        </style>
    </head>
    <body>
        <div class="widget-container">
            <h3 class="widget-title">Custom Server Information</h3>
            <div class="widget-content" id="widget-content">
                <p>Loading server information...</p>
            </div>
        </div>

        <script>
            // Widgets automatically receive context
            // Access via window.FeatherPanel.widgetContext or data attributes

            async function loadWidgetData() {
                // Get server UUID from context (automatically provided)
                const serverUuid = window.FeatherPanel?.widgetContext?.serverUuid;

                if (!serverUuid) {
                    document.getElementById('widget-content').innerHTML = '<p>No server context available</p>';
                    return;
                }

                try {
                    // Make API call to your plugin endpoint
                    const response = await fetch(`/api/user/servers/${serverUuid}/addons/my_plugin/data`);
                    const result = await response.json();

                    if (result.success) {
                        // Update widget content
                        document.getElementById(
                            'widget-content'
                        ).innerHTML = `<p>Custom data: ${result.data.message}</p>`;
                    }
                } catch (error) {
                    console.error('Failed to load widget data:', error);
                    document.getElementById('widget-content').innerHTML = '<p>Error loading data</p>';
                }
            }

            // Initialize widget when loaded
            window.addEventListener('DOMContentLoaded', loadWidgetData);
        </script>
    </body>
</html>
```

### Widget Context

Widgets automatically receive context information based on the page they're displayed on:

#### Available Context Variables

-   **`userUuid`** - Current user's UUID (available on all pages)
-   **`serverUuid`** - Current server's UUID (available on server pages only)

These are accessible through `window.FeatherPanel.widgetContext`:

```javascript
const context = window.FeatherPanel?.widgetContext || {};
const userUuid = context.userUuid;
const serverUuid = context.serverUuid;

if (serverUuid) {
    // Make server-specific API call
    fetch(`/api/user/servers/${serverUuid}/addons/my_plugin/data`);
}
```

```html
                color: #fafafa;

        </style>

    <body>
        <div class="server-info-widget">
            <h3 class="widget-title">Custom Server Metrics</h3>

            const serverUuid = window.FeatherPanel?.widgetContext?.serverUuid;

                    .then((data) => {

        </script>

```

## Best Practices

### 1. Widget Design

-   **Match FeatherPanel's design language** - Use similar colors, spacing, and typography
-   **Keep widgets focused** - Each widget should serve a specific purpose
-   **Responsive design** - Ensure widgets work well on mobile devices
-   **Loading states** - Show loading indicators while fetching data
-   **Error handling** - Gracefully handle API failures

### 2. Performance

-   **Minimize external dependencies** - Only load libraries you actually need
-   **Lazy load data** - Fetch widget data after the widget is visible
-   **Cache data** - Cache API responses when appropriate
-   **Optimize assets** - Minimize CSS and JavaScript in widget files

### 3. Context Awareness

-   **Check for context** - Always verify `serverUuid` exists before making server-specific calls
-   **Handle missing context** - Gracefully handle cases where context isn't available
-   **User permissions** - Respect user permissions in your widget logic

### 4. Widget Placement

-   **Choose appropriate locations** - Place widgets where they add the most value
-   **Consider user flow** - Don't interrupt the user's workflow unnecessarily
-   **Use priorities** - Order widgets logically using priority values

### 5. API Integration

-   **Use plugin endpoints** - Make API calls to your plugin's endpoints (e.g., `/api/user/servers/{uuid}/addons/{plugin_id}/...`)
-   **Error handling** - Always handle API errors gracefully
-   **Authentication** - Widget requests automatically include user authentication

## Troubleshooting

### Widget Not Appearing

-   Verify `widgets.json` is in `Frontend/` directory
-   Check that `component` path is correct (relative to `Components/` directory)
-   Ensure `page` identifier matches exactly (case-sensitive)
-   Verify `location` is valid for the target page
-   Check browser console for JavaScript errors
-   Ensure widget is enabled (`enabled: true`)

### Context Not Available

-   Verify you're accessing context correctly: `window.FeatherPanel?.widgetContext`
-   Server context only available on server pages
-   Check that the page actually provides context for widgets

### Styling Issues

-   Ensure your CSS doesn't conflict with FeatherPanel's styles
-   Use specific class names to avoid conflicts
-   Test on different screen sizes
-   Check browser DevTools for CSS conflicts

### API Calls Failing

-   Verify endpoint paths are correct
-   Check network tab in browser DevTools
-   Ensure backend endpoints are properly registered
-   Verify authentication tokens are being sent

## Plugin Rendered Pages

**Plugin rendered pages** (pages created via `sidebar.json` with `component` property) **do not support widgets** directly. Widgets are only available on FeatherPanel's native pages.

If you need to add widgets to a plugin page, consider:

1.  **Creating a native FeatherPanel page** that integrates your plugin functionality
2.  **Using plugin widgets** on related native pages (e.g., add widgets to `server-console` that link to your plugin page)
3.  **Embedding widget-like content** directly in your plugin HTML component
